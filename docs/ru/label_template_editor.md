# Редактор шаблонов этикеток

## Переменные
Чтобы вставить системную переменную в шаблон, используйте двойные фигурные скобки и имя переменной, например, `{{нетто_г}}`. Так же вставить переменную можно с помощью меню редактора: `Вставить > Вставить шаблон`

### Системные переменные
Значение системных переменных вычисляется в процессе работы оборудования и динамически обновляется в зависимости от текущего состояния оборудования и данных, вводимых пользователем или считанных с подключенного оборудования или датчиков.

Список системных переменных:
* `нетто_г` - Масса нетто продукта или упаковки продуктов в граммах.
* `нетто_кг` - Масса нетто продукта или упаковки продуктов в килограммах.
* `брутто_г` - Масса брутто продукта или упаковки продуктов в граммах.
* `брутто_кг` - Масса брутто продукта или упаковки продуктов в килограммах. 
* `тара_г` - Масса тары продукта или упаковки продуктов в граммах.
* `тара_кг` - Масса тары продукта или упаковки продуктов в килограммах.
* `время` - Текущее системное время. Формат времени можно редактировать, смотри ниже.
* `прод_ид` - Системный идентификатор продукта. Целочисленное значение. Автоматически увеличивается на 1 для каждого нового продукта.
* `прод_назв` - Название продукта. Строка, введенная пользователем в меню продукта.
* `прод_ид_внутр` - Идентификатор продукта. Строка, введенная пользователем в меню продукта.
* `упак_ид` - Системный идентификатор упаковки. Целочисленное значение. Автоматически увеличивается на 1 для каждой новой упаковки.
* `упак_кол_прод` - Количество продуктов в упаковке. Целочисленное значение. 
* `парт_ид` - Системный идентификатор партии. Целочисленное значение. Автоматически увеличивается на 1 для каждой новой партии.
* `парт_номер` - Номер партии. Строка, введенная пользователем в меню партии.
* `парт_назв` - Название партии. Строка, введенная пользователем в меню партии.
* `обор_ид` - Идентификатор оборудования. Строка, введенная пользователем в меню настроек оборудования.

### Пользовательские переменные
Значение пользовательской переменной вставляется в шаблон по ключу в двойных фигурных скобках.
Пользователь может создать переменную двух типов:
* `статическая` - Значение переменной не меняется после создания.
* `динамическая` - Значение переменной может меняться в процессе работы оборудования.

Для создания динамической переменной, в значение переменной нужно вставить другую переменную. Например, `номер_партии` = `Партия #{{парт_номер}}`

Значение пользовательской переменной вставляется в шаблон по ключу в двойных фигурных скобках. Например, `{{номер_партии}}`

> Все пользовательские переменные имеют тип `строка` (по умолчанию), `дата` или `дата_время`(см. ниже). 

> Значения переменных могут быть преобразованы и отформатированы с помощью функций.

#### Переменные типа `дата` и `дата_время`
Пользовательские переменные типа `дата` задаются в формате `год-месяц-день` (`yyyy-MM-dd`). Например, 1 Января 2025 года задается как `2025-01-01`.

Пользовательские переменные типа `дата_время` задаются в формате `год-месяц-день час:минуты:секунды` (`yyyy-MM-dd HH:mm:ss`). Например, 13:00 1 Января 2025 года задается как `2025-01-01 13:00:00`.


### Счетчики
Для создания счетчика, нужно указать минимальное и максимальное значение. При пересечении максимального значения, значение счетчика сбрасывается на минимальное заданное.
Как и значение переменной, значение счетчика вставляется в шаблон по ключу в двойных фигурных скобках. 

## Форматирование переменных
К системным и пользовательским переменным можно применять функции преобразования и форматирования.

Например, используя системную переменную `время` можно сформировать срок годности: `Годен до = {{date время 'dd.MM.yyyy' offset=20}}`. Результатом будет текущая дата + 20 дней в формате `dd.MM.yyyy`

### Числовые переменные

Для выполнения математических операций и форматирования числовых результатов в шаблонах используется вспомогательная функция `{{math}}`.

#### Описание
Функция `{{math}}` позволяет вычислять математические выражения, используя переменные из текущего контекста шаблона. Вычисления производятся с точностью 16 знаков (согласно IEEE 754R), а результат может быть отформатирован с помощью специального параметра.

#### Основные возможности
1.  **Математические операции**:
    *   Сложение: `+`
    *   Вычитание: `-`
    *   Умножение: `*`
    *   Деление: `/`
    *   Остаток от деления: `%`

2.  **Использование переменных**:
    Все числовые переменные, доступные в контексте, могут быть использованы внутри выражения по их именам. При этом значения автоматически преобразуются в числа (поддерживается замена запятой на точку).

3.  **Форматирование результата**:
    Параметр `format` позволяет задать маску для вывода числа. По умолчанию используется формат `#.##`.

#### Примеры использования

**1. Простые арифметические действия**
```
{{math "10 + 5"}} 
Выведет: 15

{{math "10 / 3"}} 
Выведет: 3.33 (по умолчанию #.##)
```

**2. Использование переменных из контекста**
Если в данных переданы переменные `нетто_гр = 100` и `упак_кол_прод = 3`:
```
{{math "нетто_гр * упак_кол_прод"}} 
Выведет: 300
```

**3. Форматирование результата**
Использование параметра `format` для управления количеством знаков после запятой:
```
{{math "10 / 3" format="#.####"}} 
Выведет: 3.3333

{{math "5000" format="#,###.00"}} 
Выведет: 5 000.00

{{math "5000.23" format="#,###.00"}} 
Выведет: 5 000.23

{{math "5000.23" format="00000.000"}} 
Выведет: 05000.230

{{math "45" format="00000"}} 
Выведет: 00045
```

**4. Вычисление остатка от деления**
```
{{math "10 % 3"}}
Выведет: 1
```

> **Примечание**: Если выражение не может быть вычислено (например, при синтаксической ошибке), в шаблон будет выведено сообщение об ошибке. 

> Все вычисления производятся с использованием 64-битных чисел с плавающей запятой и округляются по правилу `округление половины вниз`.

### Дополнение к инструкции `{{math}}`: Детальное форматирование чисел

Функция `{{math}}` использует стандарт Java `DecimalFormat`. Это позволяет гибко управлять отображением чисел, разделителями и специальными символами.

### Основные символы форматирования:
*   `0` — **Обязательная цифра**. Если цифр меньше, чем нулей в маске, будет выведен «0».
*   `#` — **Необязательная цифра**. Если цифры нет, на этом месте ничего не выводится (используется для подавления лишних нулей).
*   `.` — **Разделитель дробной части**.
*   `,` — **Разделитель групп разрядов** (например, для разделения тысяч). В итоговой строке будет использоваться символ, соответствующий системной локали (обычно неразрывный пробел или запятая).
*   `%` — **Процент**. Умножает значение на 100 и добавляет знак процента.
*   `¤` (`\u00A4`) — **Символ валюты**.

### Расширенные примеры:

**1. Ограничение знаков и дополнение нулями**
```
{{math "5.5" format="00.00"}} 
Выведет: 05.50

{{math "5.5" format="##.##"}} 
Выведет: 5.5
```

**2. Разделение тысяч**
```
{{math "1234567.89" format="#,###.00"}} 
Выведет: 1 234 567.89 (зависит от локали)
```

**3. Проценты**
```
{{math "0.123" format="#.0%"}} 
Выведет: 12.3%
```

## Форматирование даты и времени

Для работы с датой и временем в шаблонах используется вспомогательная функция `{{date}}`. Она позволяет форматировать существующие значения, выводить текущую дату и производить простые манипуляции с временными интервалами.

### Описание
Функция `{{date}}` принимает объект даты или строку и преобразует её в нужный формат. Если входное значение не указано, используется текущая дата.

### Основные возможности
1.  **Поддерживаемые типы входных данных**:
    *   `дата`
    *   `дата_время`
    *   `строка` (поддерживаются форматы `yyyy-MM-dd HH:mm:ss` и стандартный ISO формат даты `yyyy-MM-dd`)

2.  **Форматирование**:
    Параметр `format` (или первый позиционный аргумент) определяет маску вывода.
    *   По умолчанию для дат: `dd.MM.yyyy`
    *   По умолчанию для времени: `dd.MM.yyyy HH:mm:ss`

3.  **Смещение даты**:
    Параметр `offset` позволяет добавить или вычесть определенное количество дней из указанной даты.

### Примеры использования

**1. Текущая дата**
Если не передавать значение, выведется текущая системная дата.
```
{{date}} 
Выведет: 10.05.2024
```

**2. Форматирование конкретного значения**
Если в контексте есть переменная `ДАТА_ПРОИЗВОДСТВА`:
```
{{date ДАТА_ПРОИЗВОДСТВА format="yyyy-MM-dd"}}
Выведет: 2024-05-10
```

**3. Использование смещения (offset)**
Полезно для вычисления дат "вчера", "завтра" или через N дней.
```
Завтрашняя дата = {{date offset=1}}

Вчерашняя дата от переменной ДАТА_ПРОИЗВОДСТВА = {{date ДАТА_ПРОИЗВОДСТВА offset=-1}}
```

**4. Обработка строковых значений**
Если дата передана как строка:
```
{{date "2026-05-10 15:30:00" format="HH:mm"}}
Выведет: 15:30
```

> **Примечание**: Если передано пустое значение (`null`), Helper автоматически подставит текущую дату. Если передана строка в неподдерживаемом формате, будет выведено сообщение об ошибке парсинга или исходная строка. 

> Все операции по добавлению дней (`offset`) выполняются до применения форматирования.

### Дополнение к инструкции `{{date}}`: Детальное форматирование дат

Функция `{{date}}` использует Java `DateTimeFormatter`. Регистр символов в маске имеет критическое значение.

#### Основные символы форматирования:
*   **Год**:
    *   `yyyy` — полный год (2024)
    *   `yy` — последние две цифры (24)
*   **Месяц**:
    *   `MM` — числовой формат (05)
    *   `MMM` — сокращенное название (мая)
    *   `MMMM` — полное название (май)
*   **День**:
    *   `dd` — день месяца (01, 10)
    *   `d` — день месяца без ведущего нуля (1, 10)
    *   `EEEE` — день недели (воскресенье)
    *   `E` — краткий день недели (вс)
*   **Время**:
    *   `HH` — часы в 24-часовом формате (00-23)
    *   `hh` — часы в 12-часовом формате (01-12)
    *   `mm` — минуты
    *   `ss` — секунды
    *   `a` — маркер AM/PM

#### Расширенные примеры:

**1. Текстовое представление**
```
{{date "2024-05-10" format="d MMMM yyyy, EEEE"}}
Выведет: 10 мая 2026, воскресенье
```

**2. Время с маркером AM/PM**
```
{{date "2026-05-10 15:30:00" format="hh:mm a"}}
Выведет: 03:30 PM
```

**3. Короткие форматы**
```
{{date format="dd/MM/yy"}}
Выведет: 10/05/24
```

> **Важно**: Не путайте `MM` (месяцы) и `mm` (минуты), а также `HH` (24-часовой формат) и `hh` (12-часовой). Ошибки в регистре приведут к некорректному отображению времени или даты.
